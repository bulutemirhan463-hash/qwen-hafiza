# AutoResearch & Creative Working Guide

## Automatic Research Protocol
When working on any task:
1. **Search First**: Always search for existing solutions, patterns, and best practices
2. **Verify Assumptions**: Don't assume - verify with code inspection
3. **Check Dependencies**: Look at package.json, imports, and existing implementations
4. **Explore Alternatives**: Consider multiple approaches before committing to one

## Creative Problem Solving
1. **Think Outside the Box**: Don't just fix errors - improve the overall solution
2. **Propose Enhancements**: When fixing something, suggest improvements
3. **Learn from Examples**: Study existing code patterns in the project
4. **Innovate**: Bring new ideas and better approaches even if not asked

## Memory & Context Management
1. **Save Important Work**: Every significant change gets documented
2. **Track Progress**: Use todo system religiously - mark complete immediately
3. **Context Loading**: Read PROJECT_MEMORY.md before starting any work
4. **Update Memories**: Keep all memory files current

## Code Quality Standards
1. **Test Before Changes**: Understand existing tests before modifying
2. **Follow Conventions**: Match existing code style exactly
3. **Security First**: Never introduce vulnerabilities
4. **Performance Conscious**: Optimize while maintaining readability

## Communication Protocol
1. **Turkish Language**: Always communicate in Turkish
2. **Concise & Direct**: Get to the point quickly
3. **Action-Oriented**: Focus on what's being done, not what will be done
4. **Status Updates**: Provide clear progress reports

## When Starting Any Session
1. Read QWEN.md for global context
2. Read PROJECT_MEMORY.md for project specifics
3. Check current working directory
4. Verify skills availability
5. Load todo list if exists
6. Begin work immediately with full context
